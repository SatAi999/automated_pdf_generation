# Automated PDF Generation Pipeline using n8n and Puppeteer

This repository contains an automated print publishing pipeline that transforms raw HTML document exports (such as those generated by Notion) into publication-ready, premium A4 PDFs. The system was designed as a production automation solution for formatting, styling, and generating study notes for Anuj Jindal educational materials, ensuring strict compliance with brand guidelines.

---

## Project Overview

While modern document editing platforms (like Notion) are excellent for drafting and collaborating on text, their standard HTML exports are not optimized for print layout media. They lack pagination support, page headers and footers, dynamic page numbering, centered background watermarks, and structured column formatting. 

To bridge this gap, this project implements a rendering pipeline orchestrated by an n8n workflow. The workflow fetches raw content, runs a custom Node.js script that restructures the HTML Document Object Model (DOM) in memory, applies a tailored print stylesheet, and prints the document to PDF using Puppeteer.

---

## Design and Layout Architecture

To replicate the premium newsletter visual style of the Design Reference PDF while maintaining the layout integrity of textbook study notes, this project implements a hybrid document flow model.

### The Problem with Pure Multi-Column Layouts
A standard CSS two-column layout (`column-count: 2`) breaks down when mixed with wide elements such as diagrams, flowcharts, or comparison tables. If these elements are set to span both columns (`column-span: all`), the rendering engine balances the columns immediately preceding the spanning element. If a diagram is pushed to the next page due to its height, the balanced text before it is squeezed into a tiny vertical space, leaving the rest of the page completely empty. This results in disjointed page flows and massive empty gaps (half-pages) throughout the document.

### The Hybrid Layout Solution
Instead of forcing the entire document body into a multi-column container, this pipeline uses a custom DOM pre-processing script to wrap text elements selectively:
*   The parent page body container behaves as a standard single-column block.
*   The script groups consecutive text elements, lists, and subheadings into independent two-column containers (`.two-columns`).
*   Large diagrams, tables, and major section banners (`h1`) are kept outside these containers as full-width elements.
*   This approach lets paragraphs and lists flow in two columns naturally, while wide charts and tables stretch across the full width of the page. Because columns are only balanced within small text segments, it completely eliminates empty page gaps and page-splitting bugs.

## Brand Specifications

The pipeline strictly applies the following brand visual guidelines:
*   **Color Palette**: Strictly utilizes Primary Blue (`#1B71AC`) for sub-headings and table headers, and Brand Green (`#2AB573`) for section banners and page number backgrounds.
*   **Typography**: Imports and sets Montserrat (for headings) and Inter (for body copy) via Google Fonts.
*   **Running Headers**: Displays the full logo (`LOGO-FULL-01`) on the left and the uppercase subject/chapter meta (`ECONOMIC AND SOCIAL ISSUES | ECONOMIC GROWTH AND DEVELOPMENT`) on the right.
*   **Running Footers**: Prints the customer support phone number on the left, the website URL in the center, and the page number inside a green rounded badge on the right.
*   **Watermark**: Renders the crop logo (`LOGO-CROP`) exactly centered in the background of each page, scaled to `550px` by `550px` at exactly `20%` opacity (`opacity: 0.2;`).

## Technical Pipeline Workflow

The automation pipeline consists of the following steps:

### 1. Asset Fetching and Caching
To ensure the PDF rendering engine is robust and works reliably without network dependencies during compile time, the script downloads the remote header logo and watermark logo. It saves them locally as PNG files and encodes them into Base64 Data URIs. This guarantees that Puppeteer can resolve and render the images in its isolated print context.

### 2. Stylesheet Injection
The script parses the raw HTML file and replaces the default exported Notion styles with a customized print stylesheet. This stylesheet configures page dimensions, margins, running header/footer spacing, and custom borders for callout boxes (Knowledge Nuggets).

### 3. DOM Restructuring (Puppeteer Context)
Once the HTML is loaded into the headless browser instance, the script runs an evaluation script in the browser context:
*   It flattens the DOM by unwrapping Notion's nested `<div style="display:contents">` elements.
*   It loops through the flat children and groups paragraphs (`p`), list blocks (`ul`, `ol`), subheadings (`h2`, `h3`), blockquotes, and callouts into `.two-columns` wrappers.
*   It leaves `h1` banners, tables, and figures untouched.
*   It updates pushpin emoji markers (`📌`) in callout boxes to the branded rocket icon (`🚀`).
*   It applies a maximum height limit (`max-height: 180mm`) and object containment (`object-fit: contain`) to all figures, forcing tall diagrams to scale down to fit on a single page instead of splitting across pages.

### 4. PDF Compilation
Puppeteer prints the restructured page using the Chrome print-to-pdf media interface, passing the base64-encoded header and footer templates, setting margins to `65px` top/bottom, and exporting the final binary document.

## Installation and Environment Setup

### 1. Install Node.js Dependencies
Initialize npm and install the core Puppeteer library:
```bash
npm install puppeteer-core --save-dev
```

### 2. System Browser Configuration
The script is configured to look for Google Chrome or Microsoft Edge in standard Windows installation folders:
*   `C:\Program Files\Google\Chrome\Application\chrome.exe`
*   `C:\Program Files (x86)\Google\Chrome\Application\chrome.exe`
*   `C:\Users\Satwik\AppData\Local\Google\Chrome\Application\chrome.exe`
*   `C:\Program Files\Microsoft\Edge\Application\msedge.exe`
Ensure one of these browsers is installed on the host machine.

## Running the Pipeline

### Local CLI Execution
You can run the script manually from the command line:
```bash
node generate_pdf.js
```
This is the recommended way to test and preview changes during template development.

### n8n Automation
1. Launch the local n8n instance:
   ```bash
   npx n8n start
   ```
2. Open n8n in your web browser (default: `http://localhost:5678`).
3. Create a new workflow, click the settings menu, and select **Import from File**. Import the `workflow.json` file.
4. Click **Execute Workflow**. The workflow will invoke the local Node script and retrieve the generated PDF as a binary buffer, making it ready for downstream nodes (such as email senders, cloud storage uploads, or API responses).
